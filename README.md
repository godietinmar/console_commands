## 📋 Formato de los Archivos

Cada archivo de comandos sigue una **estructura consistente y profesional**:

```markdown
# [Nombre de la Herramienta/Base de Datos]

## Herramientas de Línea de Comandos
### [nombre_herramienta]
**Descripción:** [Qué hace la herramienta]
**Sintaxis básica:**
```bash
comando [opciones] [argumentos]
```

## [Categoría de Comandos]
### [Subcategoría]

```bash
# Descripción del comando
comando -opcion1 valor

# Ejemplo más complejo
comando -opcion1 valor -opcion2 "valor con espacios"
```

## Ejemplos de Uso Común
### [Caso de uso específico]
[Explicación del escenario]

```bash
# Serie de comandos para el caso de uso
comando1
comando2
comando3
```

## Scripts de Utilidad
[Scripts bash útiles y automatizaciones]
```

## 🤝 Contribuciones

¡Las contribuciones son **muy bienvenidas**! Este es un proyecto colaborativo para la comunidad de desarrolladores.

### Cómo Contribuir

1. **Fork** el repositorio
2. **Crea una rama** para tu contribución: `git checkout -b feature/nueva-herramienta`
3. **Mantén el formato** consistente con los archivos existentes
4. **Proporciona ejemplos** claros y útiles
5. **Verifica la precisión** de todos los comandos
6. **Organiza** los comandos en la categoría adecuada
7. **Haz commit** de tus cambios: `git commit -m 'Agregar comandos de [herramienta]'`
8. **Push** a la rama: `git push origin feature/nueva-herramienta`
9. **Abre un Pull Request**

### Estándares de Calidad

- ✅ **Formato consistente** con la estructura establecida
- ✅ **Ejemplos reales** y casos de uso prácticos
- ✅ **Comandos verificados** y funcionales
- ✅ **Documentación clara** con explicaciones detalladas
- ✅ **Organización lógica** por categorías y subcategorías
